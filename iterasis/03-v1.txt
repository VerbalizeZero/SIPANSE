Implementasi mengikuti pola Laravel MVC standar:

  name: tu.siswa.index
  fungsi: tampilkan halaman data siswa

- GET /tu/siswa/template
  name: tu.siswa.template
  fungsi: download template CSV berbasis kolom tabel siswas

  name: tu.siswa.import
  fungsi: import CSV/TXT

  name: tu.siswa.update

- DELETE /tu/siswa/{siswa}
  name: tu.siswa.destroy
  fungsi: hapus data satu siswa


D. DETAIL LOGIKA CONTROLLER
---------------------------
File: app/Http/Controllers/Tu/SiswaImportExportController.php

1) index()
- Query: Siswa::query()->latest()->get()
- Output: view('tu.siswa.index', compact('siswas'))
- Tujuan: menampilkan data terbaru paling atas

2) downloadTemplate()
- Ambil kolom tabel aktif: Schema::getColumnListing('siswas')
- Exclude kolom sistem: id, created_at, updated_at
- Return text/csv dengan header kolom
- Manfaat: template tidak hardcoded, otomatis sinkron dengan schema

3) import(Request $request)
Tahapan:
- Validasi file wajib + mimes csv/txt
- fopen file, baca header dengan fgetcsv
- Validasi header harus memuat kolom wajib: nisn, nama_siswa
- Loop setiap row:
  - skip row jika jumlah kolom != jumlah header
  - gabungkan header+row via array_combine
  - validasi nisn/nama_siswa tidak kosong
  - normalisasi tanggal_lahir dengan normalizeDate()
  - normalisasi jenis_kelamin dengan normalizeJenisKelamin()
  - trim/nullable field lain dengan nullableTrim()
- Simpan via Siswa::updateOrCreate(['nisn' => $nisn], $payload)
- Jika ada exception, kembalikan error baris import

Alasan updateOrCreate:
- NISN dipakai sebagai natural key sinkronisasi
- menghindari duplikasi record saat import ulang

4) update(Request $request, Siswa $siswa)
- Validasi input berdasarkan batas schema:
  - nisn max 20 + unique ignore current id
  - no_hp_ortu max 20
  - email_ortu harus format email
  - tanggal_lahir harus date
- jenis_kelamin dinormalisasi ke format DB (L/P)
- update record siswa

5) destroy(Siswa $siswa)
- delete record
- redirect ke index

6) Helper internal
- nullableTrim(mixed $value):
  membersihkan whitespace/quote/NBSP; string kosong jadi null

- normalizeDate(mixed $value):
  menerima beberapa format tanggal:
  - YYYY-MM-DD
  - DD/MM/YYYY
  - serial number Excel
  output final: Y-m-d atau null jika invalid

- normalizeJenisKelamin(mixed $value):
  mapping variasi nilai input menjadi 1 karakter:
  - Laki-laki, laki, male, pria -> L
  - Perempuan, female, wanita -> P
  fallback: ambil huruf pertama uppercase


E. DETAIL UI/VIEW (BLADE)
-------------------------
File: resources/views/tu/siswa/index.blade.php

Bagian utama:
1) Header action
- Tombol "Template Data" -> route('tu.siswa.template')
- Tombol "+ Tambah Siswa" -> membuka modal upload

2) Tabel data siswa
- Sumber data: variabel $siswas dari controller index()
- Kolom jenis_kelamin ditampilkan user-friendly:
  - L -> Laki-laki
  - P -> Perempuan

3) Modal upload
- Form POST multipart ke route('tu.siswa.import')
- Input file menerima .csv,.txt
- Jika error validasi file, modal upload auto-open kembali

4) Modal edit (per row)
- Form POST + spoof method PUT ke route('tu.siswa.update', $siswa)
- Pre-filled dari data siswa yang dipilih
- jenis_kelamin menggunakan select value L/P agar sesuai schema

5) Modal delete (per row)
- Konfirmasi sebelum submit DELETE ke route('tu.siswa.destroy', $siswa)

6) Script modal
- data-modal-open: buka modal berdasarkan id
- data-modal-close: tutup modal berdasarkan id
- handler generik dipakai untuk semua modal di halaman

G. KONTRAK SCHEMA DB (BATAS INPUT)
----------------------------------
File: database/migrations/2026_02_10_113000_create_siswas_table.php
Kolom penting:
- nisn -> string(20), unique
- jenis_kelamin -> string(1), nullable
- no_hp_ortu -> string(20), nullable
- tanggal_lahir -> date, nullable

Implikasi langsung ke aplikasi:

H. INSIDEN ERROR DAN AKAR MASALAH
---------------------------------

Akar masalah:

Perbaikan:

- ubah input edit menjadi select L/P
- sesuaikan validasi panjang field mengikuti schema
